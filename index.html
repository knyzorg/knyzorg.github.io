Principles

Things I keep in mind while working

Explicitness over Implicitness

Systems of all kinds should be explicit about their functions and uses regardless if it seems "obvious" or "intuitive". To other people, it might not be that way. Everything should be designed such that a partial understanding of a system would be enough to work on relevant details.

Perceived Performance above Numbers

As dozens of studies show, users care more about how fast something feels rather than how fast it actually is. The key to a good user experience is a smooth UI.

Progressive Enchancement

If it does not need JavaScript, it should not rely on it. Websites needing 20MB to display text and images is unacceptable. The web does not need more bloat.
